## Query Cookbook

All examples assume `cwd` is the repo root. Replace `<X>` with your query string.

```bash
# Find a Go symbol by name (file:line):
rg '"name":"<Name>"' .codegraph/symbols.jsonl

# Find every caller of an exported func or method:
rg '"to":"<pkg/path>.<Name>"' .codegraph/refs.jsonl                # func
rg '"to":"<pkg/path>.<Recv>.<Method>"' .codegraph/refs.jsonl       # method

# Look up a CRD's wiring (validator, config struct, template, tests):
rg -A 12 '## <CRDName>' .codegraph/domain-map.md

# Find NGINX templates defining or including a named block:
rg '"<block-name>"' .codegraph/templates.json

# Find where a Helm value is referenced:
rg -F '"<dotted.values.key>"' .codegraph/helm.json

# Find Python tests using a fixture or marker:
rg -F '"<fixture-or-marker>"' .codegraph/python-tests.json

# Find an example manifest by Kubernetes kind:
rg -F '"kind":"<Kind>"' .codegraph/examples.json

# Discover a package's files and exports:
rg -A 50 '^## <import-path>$' .codegraph/files.md
```

## Notes

- `refs.jsonl` only contains intra-repo edges. Cross-module calls (`k8s.io/...`, `nginx-plus-go-client`) are deliberately omitted.
- `pkg/client/**`, `**/zz_generated*.go`, `**/fake/**`, and `**/__snapshots__/**` are excluded from the Go index by design.
- `domain-map.md` is heuristic (name match). Use `hack/codegraph/overrides.yaml` to patch gaps when adding a CRD field.
- Regenerate with `make codegraph`. CI runs `make codegraph-check` and fails PRs with stale indexes.
