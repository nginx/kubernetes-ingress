## Query Cookbook

Prefer the `nic-graph` CLI — it returns typed JSON and is faster than grepping the raw artifacts. All examples assume `cwd` is the repo root.

```bash
# Find a Go symbol by name (file:line):
nic-graph find-symbol <Name>                              # add -kind=struct|func|method|…

# Find every caller of an exported func or method:
nic-graph find-callers <importPath>.<Name>                # plain func
nic-graph find-callers <importPath>.<Recv>.<Method>       # method

# Outgoing calls from a symbol:
nic-graph find-callees <importPath>.<Name>

# Look up a CRD's wiring (validator, config struct, template, tests):
nic-graph crd-relations <CRDName>

# Find NGINX templates by define name or filename glob:
nic-graph find-template -define=<block-name>
nic-graph find-template -glob='nginx-plus*.tmpl'

# Find where a Helm value is referenced:
nic-graph find-helm-value <dotted.values.key>

# Find Python tests by name (case-insensitive) or marker:
nic-graph find-python-test <query> -marker=<marker>

# Find example manifests by Kubernetes kind:
nic-graph find-example <Kind> -group=<subdir>

# Discover a package's files and exports:
nic-graph list-package <importPath>

# Sanity check / dependency report:
nic-graph stats
nic-graph doctor          # check go/python3/jq, -install adds missing
```

## Notes

- `refs.jsonl` only contains intra-repo edges. Cross-module calls (`k8s.io/...`, `nginx-plus-go-client`) are deliberately omitted.
- `pkg/client/**`, `**/zz_generated*.go`, `**/fake/**`, and `**/__snapshots__/**` are excluded from the Go index by design.
- `domain-map.md` is heuristic (name match). Use `hack/nic-graph/overrides.yaml` to patch gaps when adding a CRD field.
- Regenerate with `make nic-graph`. CI runs `make nic-graph-check` and fails PRs with stale indexes.
